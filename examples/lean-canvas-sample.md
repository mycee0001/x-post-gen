## 1. PROBLEM (課題)

- 製造業の現場では紙の図面・手書き作業指示書・点検記録が依然として大量に残存し、電子化が進まない
- ベテラン技能者の退職が加速(2025年問題)し、暗黙知と現場ノウハウの継承が断絶しつつある
- 既存の手書きOCRは汎用的すぎて、図面記号・寸法・専門用語の認識精度が不足
- 中小製造業は大手ベンダーのスイート製品を入れる体力がなく、DX の入り口で止まっている

## 2. CUSTOMER SEGMENTS (顧客セグメント)

- 従業員 50〜500 名規模の中堅製造業(機械加工・金属プレス・成型)
- 設計・生産技術部門の DX 推進担当
- 図面・作業指示書を年数万枚レベルで扱う現場

## 3. UVP (独自価値提案)

- 製造業の図面・作業指示書に **特化した手書きOCR + 図面記号認識** で、汎用OCRの 2 倍以上の読み取り精度
- スモールスタート ($99/月〜) で、PoC から本番まで同じサービスで完結
- 現場の PC / タブレット 1 台から始められ、初期導入コンサル不要

## 4. SOLUTION (ソリューション)

- マルチモーダル AI による図面解釈 (寸法・記号・注記 / 手書き文字)
- 既存の PDM / ERP / BOM 管理システムへの API 連携
- 暗黙知のデジタル化: ベテラン記述のデジタル化 → 作業指示の標準化テンプレ生成
- セキュアなオンプレ/閉域 VPC 運用オプション

## 5. UNFAIR ADVANTAGE (圧倒的優位性)

- 創業者自身が製造業向けエンジニアリング出身で、現場の業務フロー理解が深い
- 大手と異なり **業界特化データセット + 個別最適** を高速回転で回せる小回り
- 現場の暗黙知をインタビューし学習データ化するパイプラインが構築済

## 6. CHANNELS (チャネル)

- X (旧Twitter) での業界知見発信 + 経営層へのリーチ
- 製造業向け展示会(JIMTOF、MF-Tokyo、スマート工場EXPO 等)
- 顧客リファラル
- 一部の製造業向けメディア(MONOist、日刊工業新聞)との連携記事

## 7. REVENUE STREAMS (収益モデル)

- 月額サブスクリプション: $99 (Starter) / $499 (Team) / カスタム (Enterprise)
- 初期設定オプション(オンプレ展開時のみ)
- API 連携アドオン

## 8. COST STRUCTURE (コスト構造)

- 主要コスト: LLM / OCR API 利用料、クラウド (AWS / GCP)、開発人件費
- 可変費率は低く、マージンは成熟段階で 70% 目標
- 顧客獲得は広告より X 運用 + 展示会 + 紹介中心

## 9. KEY METRICS (主要指標)

- MRR / ARR
- 図面 1 枚あたり処理コスト
- 新規トライアル → 本番契約の転換率
- ベテラン記述のデジタル化件数(差別化指標)
