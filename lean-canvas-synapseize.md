# Synapseize リーンキャンバス

## 1. 課題（Problem）

ADHD当事者・ADHD傾向を持つ人が日常的に直面する実行機能の障壁。

1. **実行機能の凍結（Executive Freeze）** — やるべきことは分かっていても最初の一歩が出ない。既存のタスク管理アプリは着手前の判断コストを増やすばかりで逆効果になる。
2. **時間盲（Time Blindness）** — 時間の長さや締切までの距離が掴めず、作業の見積もりと実際が乖離する。
3. **羞恥心と自己否定の蓄積** — 未完了タスクが積み上がり「自分はできない」という感覚が強化され、アプリを開くこと自体が苦痛になる。

### 代替ソリューションの欠点

- Notion / Obsidian / ClickUp：セットアップ自体が目的化し運用に移れない
- Todoist / Duolingo：通知疲労と罪悪感で離脱しやすい
- Tiimo / Owaves：ビジュアル化は得意だが「着手支援」まで届かない
- Brain.fm / Endel：受動的BGMでタスク分解や再開管理は弱い

## 2. 顧客セグメント（Customer Segments）

### アーリーアダプター

- 20〜40代の成人で**ADHD診断済み、または自覚的にADHD傾向がある人**
- 既存のタスク管理アプリを試したが継続できなかった経験がある層
- SNS（X / Instagram）でADHD関連情報を能動的に収集している層

### プライマリセグメント

- 上記に加え、スマートフォンを日常的に利用する日本国内の成人（推定315万人の成人ADHD有病者のうち、アプリ利用意向を持つ約20%＝約63万人がSAM）

### セカンダリセグメント

- ADHDの診断はないが、集中力の維持・先延ばし癖に悩む社会人・学生
- 発達障害の疑いを持ちながら未受診の層（いわゆるグレーゾーン）

### 中長期のターゲット拡張

- 企業の従業員支援プログラム（EAP）導入企業（B2B）
- 医療機関・カウンセリング事業者（B2B2C）

---

## 3. 独自の価値提案（Unique Value Proposition）

> **「頭の中の混沌をそのまま預けると、今やる1つへ整えて返す。」**

ADHD特性のあるユーザーに対し、タスク管理アプリの「使う前に整理と設定を要求する」構造を拒否し、**混沌を受け止め → AIが分解 → 今やる1つを前景化 → 集中を支援 → 失敗を責めず再開を促す**という一連の体験を、ゼロセットアップで提供する。

### ハイレベルコンセプト

「管理を増やすアプリ」ではなく、「始めやすくし、続けやすくするアプリ」。
医療機器や診断ツールではなく、日々の実行機能を支える**外部認知サポートツール**。

---

## 4. ソリューション（Solution）

各課題への対応を一つのアプリに統合する。

| 顧客課題                       | 実装済みソリューション                                                                                                                                    |
| ------------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 何から始めればよいか分からない | **Chaos Dump**（テキスト / 音声30秒以内）+ OpenAI Responses API による **AIタスク分解**（ナノステップ化・推定時間付与）                                   |
| やることが多すぎてフリーズする | **Today タブ**で「今やる1つ」を前景化、時間帯別セクション表示                                                                                             |
| 時間感覚が掴みにくい           | **Focus Session**（15/25/45/60分プリセット）、リングタイマー、休憩3/5/10分、音声ガイダンス（中盤・終了1分前）                                             |
| 途中で詰まる                   | **Flow Dive 音声相談**（録音 → STT → AI助言 → TTS再生）                                                                                                   |
| 失敗が積み上がって再開しづらい | **Archive / Revive**、**Lazarus Wash**（stale taskの自動アーカイブ）、**Momentum Bloom**（28日間の非懲罰的進捗可視化、rising/steady/restarting トレンド） |
| カレンダーとの連動             | **Google Calendar 連携**（読み取り専用、5分前プッシュ通知・事前AI分解）                                                                                   |

### 差別化を支える実装上の意思決定

- **ゼロセットアップ** — カテゴリ設定やプロジェクト構造の事前構築を不要化
- **音声ネイティブ入力** — タイピングが困難な場面でも即座に入力可能
- **非懲罰的リカバリ** — 未完了の罪悪感を生まない自動アーカイブと再開導線
- **オフラインキュー** — 通信途絶時も操作を退避し自動再送

## 5. チャネル（Channels）

### Phase 1（ローンチ〜6ヶ月）：オーガニック中心

- **App Store Optimization（ASO）** — 「ADHD タスク管理」「集中 アプリ」「先延ばし 対策」等のキーワード最適化
- **SNS発信（X / Instagram）** — ADHD当事者コミュニティ向け発信、開発ストーリーの共有
- **SEOブログ** — ADHD関連のお役立ち記事（日本語）でオーガニック流入を獲得
- **ADHD当事者会・支援団体** — コミュニティとのリレーション構築

### Phase 2（6ヶ月〜18ヶ月）：有料施策併用

- **Apple Search Ads / Google Ads** — アプリインストール広告
- **インフルエンサーマーケティング** — ADHD発信者とのコラボ
- **プレスリリース・メディア露出**

### Phase 3（18ヶ月〜）：パートナーシップ

- **医療機関・カウンセリングサービス** との連携
- **企業EAPへの組み込み提案**
- **B2B法人プラン** の展開

### 配信 / 決済チャネル

- **App Store（iOS）**、**Google Play（Android）** のアプリ内課金（IAP）
- **RevenueCat** によるサブスクリプション状態管理

## 9. 圧倒的な優位性（Unfair Advantage）

競合が短期間で模倣しにくい、構造的な差別化要素。

1. **ADHD特化の設計出発点** — 汎用タスク管理のカスタマイズではなく、実行凍結・時間盲・羞恥の3大痛みを設計の起点としているため、機能追加で後から獲得できない一貫した体験ロジックを保有。
2. **非懲罰的リカバリの独自機能** — **Lazarus Wash**（自動アーカイブ）と **Momentum Bloom**（非懲罰的進捗可視化）は既存の主要競合には見当たらない独自設計。
3. **統合型体験** — タスク分解・集中支援・進捗可視化・カレンダー連携を一つのアプリに統合し、複数ツール併用の認知負荷を根本から排除。
4. **サーバー主導の整合性と冪等性** — 変更系処理をすべて RPC / Edge Function 経由に集約し、`rpc_requests` による冪等化・RLS による本人データ保護・オフラインキューによる再送を担保。スケール時の運用コストと障害影響範囲を構造的に抑える設計。
5. **医療領域に踏み込まない位置取り** — 医療機器認証や薬事対応の重い規制を回避しつつ、「実行機能を支える外部認知サポートツール」として独自ポジションを確立。競合が医療領域に踏み込むと重い規制コストを負うのに対し、本プロダクトは機動力で優位に立つ。
6. **代表者によるフルスタック一人開発** — UI / バックエンド / AI連携 / DevOps までを一貫して設計する意思決定の速さと、外注依存度の低さによる粗利率の高さ。

## 付録：主要リスクと対策（サマリ）

事業計画書の 8 章より、特に事業運営で注視すべきリスクを抜粋。

| リスク             | 対策                                                                                                  |
| ------------------ | ----------------------------------------------------------------------------------------------------- |
| ユーザー獲得の遅れ | ASO/SEO/SNSの多角的アプローチ、KPI未達時は広告投資を前倒し                                            |
| 有料転換率の低迷   | 無料枠の制限値調整（`app.settings.free_ai_breakdown_daily_limit`）、オンボーディング最適化、A/Bテスト |
| 競合の参入・模倣   | ADHD特化の深い理解と継続的機能改善でブランドロイヤリティを構築                                        |
| OpenAI API価格改定 | モデル軽量化、プロンプト最適化、将来的なオンデバイス推論の採用検討                                    |
| 医療機器該当性     | 診断・治療・予防を行わない旨を利用規約とアプリ内に明記                                                |
| 個人情報保護       | テレメトリのPIIマスキング（`sanitizeRecord`）、Google token 暗号化、データ削除権の保証                |
