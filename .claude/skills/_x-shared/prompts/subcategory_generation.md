# lean-canvas サブカテゴリ生成プロンプト

## 目的

`x-reply` / `x-quote` の主要クエリ検索でヒット数が閾値 (デフォルト 5 件) 未満になった時のフォールバック。
lean-canvas の意味的な周辺領域に検索を広げるため、Claude が **サブカテゴリを 3 個** 生成する。

## 入力されるコンテキスト

- `lean-canvas.md` の `sections` (problem / solution / customer_segments / channels / uvp / unfair_advantage)
- `topic_tags` (主要クエリで既に使用したキーワード)

## ミッション

lean-canvas の主要トピックから **意味的に隣接** するが **角度が異なる** サブカテゴリを **ちょうど 3 個** 生成する。
各サブカテゴリは互いに重複せず、また `topic_tags` と語句レベルで完全一致しないこと。

## 制約

### 個数

- **必ず 3 個**。多すぎても少なすぎても NG (API クレジット節約のため)

### 軸 (axis)

以下の 4 軸から **3 個 (異なる軸を 3 つ)** 選んで割り当てる:

| axis | 説明 | 例 (主要トピック=「製造業DX」の場合) |
|---|---|---|
| `上位概念` | 主要トピックを包含する一段上の概念 | 「日本の製造業」「中小企業の生産性」 |
| `隣接領域` | 関連する別ジャンル / 周辺技術 | 「IoT・スマートファクトリー」「ものづくり補助金」 |
| `具体ユースケース` | 主要トピックの具体的な現場 / 機能 | 「町工場のペーパーレス化」「図面OCRの精度問題」 |
| `業界横展開` | 同じ課題構造を持つ別業界 | 「建設業の2024年問題」「物流業のDX」 |

### クエリ設計

各サブカテゴリには `queries` (1〜2 個の X 検索クエリ) を含める:

- 日本語 OR 英語キーワードを `OR` で結合 (例: `"スマートファクトリー OR IoT工場"`)
- 主要トピックのキーワードと **完全に同じ語** は使わない (重複検索を避ける)
- 過度に一般的な語 (例: 「DX」「AI」のみ) は避け、3〜10 文字の具体語を 2〜3 個含める

## 出力フォーマット

**必ず以下の JSON 配列形式で、それ以外は出力しない:**

```json
[
  {
    "name": "ものづくり補助金",
    "axis": "隣接領域",
    "queries": ["ものづくり補助金 OR IT導入補助金"],
    "rationale": "製造業DX 投資の資金源となる補助金は同じ意思決定者が情報収集している"
  },
  {
    "name": "建設業の2024年問題",
    "axis": "業界横展開",
    "queries": ["建設業 2024年問題 OR 建設DX"],
    "rationale": "技能継承・人手不足という同型の課題構造を持ち、解決策の事例が転用可能"
  },
  {
    "name": "町工場のペーパーレス化",
    "axis": "具体ユースケース",
    "queries": ["町工場 ペーパーレス OR 中小製造業 紙文化"],
    "rationale": "製造業DX の中でも最も初期段階の現場課題で、潜在顧客の生の声が拾える"
  }
]
```

## 必須チェック (生成後の自己検証)

- [ ] `subcategories` が **ちょうど 3 個**
- [ ] `axis` が 3 個とも異なる
- [ ] `queries` の語が `topic_tags` と完全一致していない
- [ ] `rationale` が空でない
- [ ] JSON が valid (キャッシュ保存スクリプトに渡される)

## 保存

生成後、ユーザーは以下のスクリプトでキャッシュに保存する:

```bash
python3 .claude/skills/_x-shared/scripts/subcategory_generator.py save \
  --canvas-hash <canvas.content_hash> \
  --subcategories-json '<上記 JSON>'
```

これにより、同じ lean-canvas (content_hash 一致) では再生成不要となる。
