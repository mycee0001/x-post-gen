# lean-canvas サブカテゴリ生成プロンプト

## 目的

`x-reply` / `x-quote` の主要クエリ検索でヒット数が閾値未満になった時のフォールバック。
lean-canvas の意味的な周辺領域に検索を広げるため、Claude が **サブカテゴリを 5 個** 生成する。

## 入力されるコンテキスト

- `lean-canvas.md` の `sections` (problem / solution / customer_segments / channels / uvp / unfair_advantage)
- `topic_tags` (主要クエリで既に使用したキーワード)

## ミッション

lean-canvas の主要トピックから **意味的に隣接** するが **角度が異なる** サブカテゴリを **ちょうど 5 個** 生成する。
各サブカテゴリは互いに重複せず、また `topic_tags` と語句レベルで完全一致しないこと。

## 制約

### 個数

- **必ず 5 個**(旧版 3 個から拡張。x-reply の活発度フィルタで取得数が減りやすいため、検索面を広げる目的)

### 軸 (axis)

以下の 5 軸から **5 個 (各軸 1 個ずつ)** 選んで割り当てる:

| axis | 説明 | 例 (主要トピック=「製造業DX」の場合) |
|---|---|---|
| `上位概念` | 主要トピックを包含する一段上の概念 | 「日本の製造業」「中小企業の生産性」 |
| `隣接領域` | 関連する別ジャンル / 周辺技術 | 「IoT・スマートファクトリー」「ものづくり補助金」 |
| `具体ユースケース` | 主要トピックの具体的な現場 / 機能 | 「町工場のペーパーレス化」「図面OCRの精度問題」 |
| `業界横展開` | 同じ課題構造を持つ別業界 | 「建設業の2024年問題」「物流業のDX」 |
| `ペルソナ拡張` | 顧客セグメントの周辺・拡張ペルソナ | 「現場リーダーの権限委譲」「ベテラン技能者の悩み」 |

### クエリ設計

各サブカテゴリには `queries` (1〜2 個の X 検索クエリ) を含める:

- 日本語 OR 英語キーワードを `OR` で結合 (例: `"スマートファクトリー OR IoT工場"`)
- 主要トピックのキーワードと **完全に同じ語** は使わない (重複検索を避ける)
- 過度に一般的な語 (例: 「DX」「AI」のみ) は避け、3〜10 文字の具体語を 2〜3 個含める

## 出力フォーマット

**必ず以下の JSON 配列形式で、それ以外は出力しない:**

```json
[
  {
    "name": "ものづくり補助金",
    "axis": "隣接領域",
    "queries": ["ものづくり補助金 OR IT導入補助金"],
    "rationale": "製造業DX 投資の資金源となる補助金は同じ意思決定者が情報収集している"
  },
  {
    "name": "建設業の2024年問題",
    "axis": "業界横展開",
    "queries": ["建設業 2024年問題 OR 建設DX"],
    "rationale": "技能継承・人手不足という同型の課題構造を持ち、解決策の事例が転用可能"
  },
  {
    "name": "町工場のペーパーレス化",
    "axis": "具体ユースケース",
    "queries": ["町工場 ペーパーレス OR 中小製造業 紙文化"],
    "rationale": "製造業DX の中でも最も初期段階の現場課題で、潜在顧客の生の声が拾える"
  },
  {
    "name": "中小企業の生産性",
    "axis": "上位概念",
    "queries": ["中小企業 生産性 OR 中小企業 業務改善"],
    "rationale": "製造業DX を含む包括テーマで、経営者層の議論が活発"
  },
  {
    "name": "現場リーダーの権限委譲",
    "axis": "ペルソナ拡張",
    "queries": ["現場リーダー OR 工場長 マネジメント"],
    "rationale": "DX 推進の実行役となる中間管理職の悩みが集まる"
  }
]
```

## 必須チェック (生成後の自己検証)

- [ ] `subcategories` が **ちょうど 5 個**
- [ ] `axis` が 5 個とも異なる (上位概念 / 隣接領域 / 具体ユースケース / 業界横展開 / ペルソナ拡張)
- [ ] `queries` の語が `topic_tags` と完全一致していない
- [ ] `rationale` が空でない
- [ ] JSON が valid (キャッシュ保存スクリプトに渡される)

## 保存

生成後、ユーザーは以下のスクリプトでキャッシュに保存する:

```bash
python3 .claude/skills/_x-shared/scripts/subcategory_generator.py save \
  --canvas-hash <canvas.content_hash> \
  --subcategories-json '<上記 JSON>'
```

これにより、同じ lean-canvas (content_hash 一致) では再生成不要となる。

---

## 第 2 段階: 既存サブカテゴリへの追加生成 (5 個 → 計 10 個)

リーチ・活発度フィルタを通過した候補が依然として 5 件未満の場合に、
第 2 段階としてサブカテゴリを **追加 5 個** 生成する。

### 入力 (第 2 段階モード)

通常の入力に加えて、**既存の 5 サブカテゴリ** が与えられる
(キャッシュ `subcategories.json` に保存済みのもの)。

### 制約 (第 2 段階モード)

- 既存 5 個の `name` / `queries` と **完全に異なる** 5 個を生成する
  (キーワード重複も避ける)
- 既存と同じ軸 (`axis`) を使ってよいが、視点 (角度) が違うこと
  - 例: 既存に「上位概念=日本の製造業」がある → 新規は「上位概念=中小企業の生産性」など
- `topic_tags` および既存サブカテゴリの `queries` の語と重複しないこと
- 必ず **5 個ちょうど**

### 出力 (第 2 段階モード)

通常モードと同じ JSON 配列形式で **5 個だけ** 出力する。
既存 5 個は含めない。マージは保存スクリプト側で自動的に行う。

### 保存 (第 2 段階モード)

`save` ではなく **`append`** を使う:

```bash
python3 .claude/skills/_x-shared/scripts/subcategory_generator.py append \
  --canvas-hash <canvas.content_hash> \
  --subcategories-json '<新規 5 個の JSON 配列>'
```

`append` は既存エントリに追加し、name 重複は自動的にスキップされる。
結果として `total_count` が 10 になる (既存 5 + 新規 5)。
