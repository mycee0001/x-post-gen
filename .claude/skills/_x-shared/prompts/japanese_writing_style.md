# 日本語文章ベストプラクティス (X 投稿系すべての共通ルール)

このガイドは `/x-post`・`/x-quote`・`/x-reply` の **すべての本文生成** に必ず適用する。
個別の生成プロンプト (post_generation.md / quote_generation.md / reply_generation.md) を読むときは、このファイルも必ず一緒に読み込み、両方の制約に従うこと。

---

## 1. 行と文の長さ

- 一行は **最大 50 文字**、理想は 35 文字前後
- 句点までが長くなりそうなら、読点の位置で改行を入れる
- 例外: カギカッコ内の引用・固有名詞、または視覚的・心理的効果を狙う場合

【悪い例】
> 昨日の打ち合わせで、来期のプロダクト戦略について議論したのだが、結局結論が出ずに次回に持ち越しとなった。

【良い例】
> 昨日の打ち合わせで来期のプロダクト戦略を議論しましたが、
> 結論が出ずに次回へ持ち越しとなりました。

---

## 2. 文体（口調）

| 用途 | 文体 | 補足 |
|---|---|---|
| x-post (発信型) | **常体（だ・である調）をベース** | 読者を意識した説明や特定のメッセージを伝えたい場合は、部分的に丁寧体を許容。**段落単位で統一** すること |
| x-quote (受動型) | **丁寧体（です・ます）** | 引用元の話者を踏み台に発信するため、読み手への目線を保つ |
| x-reply (受動型) | **丁寧体（です・ます）** | 対話なので、より柔らかく |

- 一人称は **「私」** で統一（俺・僕・自分は使わない）

---

## 3. 使用禁止用語

下記のような語彙はいかなる文脈でも使用しない。

### 略式侮蔑語

キモい、キショい、ウザい、くっさ、くさ、は？、それらの語尾「い」を抜いた感嘆形（「キモ」「ウザ」など）

### 特定グループへのレッテル

エアプ、キッズ、陰キャ、陽キャ、オタク（属性ラベルとして揶揄する用法）、チー牛

### 差別用語

ガイジ、めくら、つんぼ、池沼、アスペ、統失、クロンボ、その他障害・人種・出自への侮蔑語

### 性的侮辱を含む用語

まんさん、ちんさん、弱者男性、クソオス、フェミ（蔑称用法）など

> 特に **略式侮蔑語と「特定グループへのレッテル」は手癖で出やすい** ので強く意識して避けること。

## 4. 使う単語のレベル

### 中学生でもわかる単語に置き換える

- 専門用語・略語（CPA, KPI, ARPU, NEAT, COGS, TAM, ROAS など）は、平易な言葉に置き換える
- どうしても専門用語を使う場合は、その意味を一文で添える
- カタカナ語の多用も避ける。日本語に置き換え可能なら置き換える

【悪い例】
> CPAを下げるためにオンボーディングのCVRを最適化する。

【良い例】
> 顧客一人を獲得する費用を下げるため、初回利用までの離脱を減らす設計を見直す。

### ADHD/発達特性領域の用語の扱い

- 当事者向け文脈（x-reply / x-quote）では、ある程度の専門語（実行機能・ワーキングメモリ・ニューロダイバーシティなど）は読者層に届くため使用可
- ただし「認知資源」「メタ認知」など抽象度が高い語は、文中で何を指すかが伝わるよう一文を補う

## 5. 表現技法（優先順位順）

下記の番号は影響度の高い順。**6 → 7 → 4 → 3 → 2 → 5 → 1** の順で守る。

### 6. 主述を対応させる（最優先）

【悪い例】私の夢は、社長になりたいです。
【良い例】私の夢は、社長になることです。

### 7. 修飾語は被修飾語の直前に置く

【悪い例】私は連休中、毎朝夫が書いた新聞連載記事を読み返した。
【良い例】私は連休中、夫が書いた新聞連載記事を毎朝読み返した。

### 4. 指示語を削る・具体化する

【悪い例】それを使えば作業が早くなります。
【良い例】このツールを使えば作業が早くなります。

### 3. 接続詞を削る

【悪い例】しかし、この方法には欠点があります。なぜなら、〜
【良い例】この方法には欠点があります。〜

### 2. 不要な修飾語を削る

「とても」「非常に」「本当に」「めちゃくちゃ」などは多用しない。

### 5. 重複表現を避ける

「頭痛が痛い」「まず最初に」「危険が危ない」「収入を稼ぐ」など。

### 1. 短く言い切る

語尾を「〜と思います」「〜と感じます」「〜気がします」に頼り過ぎない。
言い切る箇所と推測の箇所を意図的に切り替える。

## 6. 引用パターンの禁止 (x-reply / x-quote 共通)

### NG パターン

リプライ・引用ツイート本文の **冒頭で必ず元ポストの一文をカギカッコで引用する** スタイルは、パターン化して不自然になる。例えば下記は禁止：

【NG】
> 『〇〇は△△だ』、当事者向けプロダクトを作る側でも本当にそう感じます。

### OK パターン

- **自分の主張が元ポストの特定の語句に明確に紐づく場合に限り**、その語句に触れる
- 触れる場合も、冒頭にカギカッコ引用を貼り付けるのではなく、自分の言葉の中に自然に織り込む
- 元ポストへの言及は **無くてもよい**。むしろ自分の観察・経験・補足から書き始める方が自然な対話になる

【良い例 1：引用なしで自分の観察から始める】
> 環境を整えるだけで集中の質が変わる、という体感はADHD当事者にも共通します。
> 物理的に視界の刺激を減らせるかが、その日の処理能力の上限を決めますよね。

【良い例 2：引用元の特定語句に明確に紐づくときだけ、自然に織り込む】
> 「気合いより仕組み」という指摘、設計の観点でも揺るぎない原則だと思います。
> 意志は脳の予算を一気に食う高コスト機能なので、頼ると持続しません。

## 7. 出力チェックリスト（自己点検）

本文を書き終えたら、最終出力前に以下を確認する：

- [ ] 冒頭がカギカッコ引用から始まっていないか（リプライ・引用 QT のみ）
- [ ] 一行 50 字超の行がないか
- [ ] 文体が用途に合っているか（post = 常体、quote/reply = 丁寧体）
- [ ] 一人称が「私」になっているか
- [ ] 禁止用語が混入していないか
- [ ] 専門用語・略語に説明が添えられているか、もしくは平易な語に置き換えられているか
- [ ] 主述が対応しているか
- [ ] 「とても」「本当に」「めちゃくちゃ」など修飾語の多用がないか
- [ ] 強い断定（「絶対」「必ず」「100%」「唯一」）がないか
